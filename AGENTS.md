# Coding style

* Keep comments minimal.
If the code is obvious it doesn't need a comment explaining it.
Comments should be for why you're doing something out of the ordinary or something
that is not obvious.
Or for code that is particularly hard to read or complex.

# .gitignore

Add any files created intermediately (e.g. `.db` files, build artifacts) to `.gitignore`. Prefer glob patterns when possible.
